Use the cvox.Api module to control ChromeVox.

API Reference: cvox.Api

Types

NodeDescription

( object )
Data structure for holding information on how to speak a particular node. The string data is separated into context, text, userValue, and annotation to enable ChromeVox to speak each of these with the voice settings that are consistent with how ChromeVox normally presents information about nodes to users.

Properties of NodeDescription

context ( string )
Contextual information that the user should hear first which is not part of main content itself. For example, the user/date of a given post.
text ( string )
The main content of the node.
userValue ( string )
Anything that the user has entered.
annotation ( string )
The role and state of the object.

Methods

isChromeVoxActive

cvox.Api.isChromeVoxActive()

Returns true if ChromeVox is currently running. If the API is available in the JavaScript namespace but this method returns false, it means that the user has (temporarily) disabled ChromeVox.

Return value - boolean

True if ChromeVox is currently active.

speak

cvox.Api.speak(string textString, number queueMode, object properties)

Speaks the given string using the specified queueMode and properties.

Parameters

textString ( string )
The string of text to be spoken.
queueMode ( optional number )
Valid modes are 0 for flush; 1 for queue.
properties ( optional object )
Speech properties to use for this utterance.

  • LISTBOX
  • LIST_ITEM
  • NEW_MAIL
  • OBJECT_CLOSE
  • OBJECT_DELETE
  • OBJECT_DESELECT
  • OBJECT_OPEN
  • OBJECT_SELECT

  • WRAP_EDGE

syncToNode

cvox.Api.syncToNode(Node targetNode, boolean speakNode)

Synchronizes ChromeVox's internal cursor to the targetNode so that when the user resumes reading, they will be starting from a reasonable position.

Parameters

targetNode ( Node )
The node that ChromeVox should be synced to.
speakNode ( optional boolean )
If true, speaks out the node. Defaults to FALSE.
